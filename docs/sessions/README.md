# RegressProof Session Notes

This directory stores short per-session notes for `RegressProof`.

Purpose:

- preserve context that would otherwise be lost with chat history
- connect work sessions to code and documentation changes
- make the next session restart fast and predictable
- keep durable project memory inside the standalone `RegressProof-cli` repository

Recommended filename format:

- `YYYY-MM-DD-session.md`

Recommended workflow:

1. Read `docs/REGRESSPROOF_INDEX.md`
2. Read `docs/REGRESSPROOF_WORKFLOW_MEMORY.md`
3. Read the latest file in `docs/sessions/`
4. At session end, add a new note using `docs/REGRESSPROOF_SESSION_TEMPLATE.md`

Do not treat old workspace exports, temporary clones, or the inactive
`RegressProof.git` remote as project memory. If something useful is found there,
move the relevant note, config, or artifact reference into this repository tree.
