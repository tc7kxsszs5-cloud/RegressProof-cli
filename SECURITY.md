# Security Policy

## Supported Scope

Security reports are welcome for the current standalone `RegressProof-cli` repository.

## Reporting a Vulnerability

Please do **not** report sensitive vulnerabilities in a public GitHub issue.

Instead, contact the repository owner through GitHub with:

- a clear description of the issue
- impact assessment
- reproduction steps
- proof-of-concept details if available
- any suggested mitigation

If the issue is low-risk or not sensitive, a normal issue may still be appropriate.

## What to Expect

Maintainers will try to:

- acknowledge the report
- reproduce the issue
- assess severity and scope
- coordinate a fix when confirmed

Response times may vary, but responsible disclosure is appreciated.
